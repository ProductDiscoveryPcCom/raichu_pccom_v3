## P3 — Bugs funcionales

### P3.1 — `min_score` ignora puntuaciones de 0 en correccion dual
**Severidad:** Alta — impacto en calidad
**Archivo:** `core/openai_client.py:346`

```python
min_score = min(claude_score, openai_score) if claude_score and openai_score else claude_score or openai_score
```

`claude_score and openai_score` evalúa False cuando cualquiera vale 0 (puntuación válida que indica "muy mal"). En ese caso cae al `or`, devolviendo el otro score en lugar del mínimo. Resultado: si Claude da 0 y OpenAI da 70, `min_score` = 70, perdiendo la señal crítica.

**Fix:** comparar contra `None` explícitamente.
```python
scores = [s for s in (claude_score, openai_score) if isinstance(s, (int, float))]
min_score = min(scores) if scores else 0
```

### P3.2 — `total_comments` siempre 0 al normalizar producto desde n8n
**Severidad:** Media — pérdida silenciosa de datos
**Archivo:** `utils/product_json_utils.py:238`

`_normalize_n8n_product()` hardcodea `"totalComments": 0` y luego `_build_product_data()` lee esa key (línea 270). Cualquier producto que venga por la ruta n8n llega al pipeline sin contador de reseñas, aunque el endpoint los exponga. Las mini-stories de reviews y los hints de credibilidad pierden señal.

**Fix:** leer `raw.get("total_comments", raw.get("totalComments", 0))` y propagarlo.

### P3.3 — `list(set(...))` pierde orden en frases IA y enlaces faltantes
**Severidad:** Baja — afecta reproducibilidad
**Archivo:** `core/openai_client.py:351, 356`

```python
all_ai_phrases = list(set(claude_ai + openai_ai))
all_missing = list(set(claude_missing + openai_missing))
```

`set()` no garantiza orden → cada ejecución produce un prompt de Stage 3 ligeramente distinto, dificultando A/B testing y reproducción de bugs.

**Fix:** `list(dict.fromkeys(claude_ai + openai_ai))` (preserva orden de inserción).

### P3.4 — Tabla de specs descarta filas con >2 columnas
**Severidad:** Media — pérdida de specs
**Archivo:** `utils/product_json_utils.py:118-119`

```python
parts = [p.strip() for p in line.split('|')[1:-1]]
if len(parts) == 2 and parts[0] and parts[0] != 'Clave':
```

Si la tabla markdown tiene una tercera columna (unidad, nota), la fila se descarta entera. Específicamente para n8n con tablas de tipo `| Clave | Valor | Unidad |`.

**Fix:** `if len(parts) >= 2 and parts[0] and parts[0] != 'Clave': specs[parts[0]] = ' '.join(parts[1:]).strip()`.

### P3.6 — Truncado fijo de 8000 chars en scraping de competidores
**Severidad:** Baja — pérdida de contexto

Trunca a 8000 chars sin considerar el tamaño objetivo del rewrite. Para competidores largos (4-5k palabras) se pierden las secciones finales (FAQs, conclusiones), justo lo que más necesita el análisis.

### P3.7 — `puntuacion_general` devuelve 0 si es dict sin las keys esperadas
**Severidad:** Baja — degradación silenciosa
**Archivo:** `core/openai_client.py:340-345`

```python
if isinstance(claude_score, dict):
    claude_score = claude_score.get('general', claude_score.get('total', 0))
```

Si Claude devuelve `{"seo": 80, "tono": 75}` (sin `general`/`total`), el score se silencia a 0. El defensive code creado en commit 6c2b89c cubre el shape común pero no logea el shape inesperado.

**Fix:** loggear `logger.warning("puntuacion_general dict shape inesperado: %s", list(claude_score.keys()))` antes del fallback a 0.

## Follow-up (siguiente sprint)

### P3.10 — Drift commit↔código en scraping paralelo
**Severidad:** Media — discrepancia de documentación
**Archivos:** `core/scraper.py`, commit 4c83b39

El commit 4c83b39 anuncia "Parallelize competitor scraping with
ThreadPoolExecutor(max_workers=4)" pero `ThreadPoolExecutor` solo
existe en `utils/serp_research.py`. `core.scraper.scrape_competitor_urls`
sigue siendo secuencial con `time.sleep(0.5)` entre requests.

**Impacto:** Scraping de N competidores tarda N × (request_time + 0.5s)
en lugar de max(request_times). Con 5 competidores: ~12s en vez de ~3s.

**Fix propuesto:** aplicar ThreadPoolExecutor a scrape_competitor_urls,
eliminar el sleep, mantener el retry/timeout actual.

**Verificación:** medir tiempo de scraping con 5 URLs antes/después.
